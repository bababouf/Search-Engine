Overview

Written in Java, this search engine will take a corpus of documents (corpus must be all .JSON or all .TXT files) and create an index. The creation of the index frontloads the processing time so that the latency between a query and response is kept low. Once the index is created, one of two modes can be selected: boolean retrieval mode or ranked retrieval mode. More information regarding each of these modes can be found upon their selection.

The option of using a default corpus of national park service documents (30,000 scraped from NPS.org websites) can be selected below instead of uploading a corpus.

Get Started

Default Directory

Choosing this option will use a default directory (built from scraping national park webpages). This allows for testing the various query modes the program has to offer.

Upload Directory

Choosing this option allows for a directory of .TXT or .JSON files to be uploaded. Note there is a limit to the size of this folder!
